# Retro: 2026-07-23

## What we shipped

- **`used-car-monitor`** (atlas-store) — Cron-driven monitor: Craigslist
  adapter, NHTSA vPIC decode, comps-style valuation, daily cron with
  Telegram alerts, 11/11 tests passing.
- **`gstack-hermes` skill port** — First 5 skills ported to Hermes
  with router; the gstack-hermes skill set loads via `skill_view`.
- **`openagents-skills` v1.0.0** — 36-skill fork of garrytan/gstack
  published to github.com/TalonForgeHQ/openagents-skills, default
  branch `main`, single-commit-history-but-now-5-commits, MIT.
- **`AUDIT.md` report** — 5 issues found and fixed (branch divergence,
  .bat URL, FORMAT.md claim, README size claim, missing .gitignore).

## What broke

- **Used-car-monitor Craigslist integration** — 200/empty body
  returned during test; sandbox HTTP middleware blocks legitimate
  Craigslist responses. Adapter works fine in real network access but
  failed during validation. Workaround: switch to file-feed ingestion
  + opt-in MCP for live sources.
- **/tmp git clone silent failure** — When install scripts clone to
  `/tmp/...` paths, MSYS + Windows sandbox silently drops the
  destination. Symptom: "Repo layout changed — could not find skills/
  directory". Operational fix: use relative paths or pre-existing
  clone dirs.
- **No automated branch protection** — First push had master + main
  diverged and master became default on GitHub with stale URLs.
  Manual fix this time; need a CI workflow.

## What we learned

- **Garbage in, semi-readable out** — porting Claude-Code-specific
  body text via auto-trim left `[PORT-REMOVED]` markers everywhere.
  Writing 11 skills fresh + 25 honest stubs was cheaper than polishing
  the broken port. Pattern: when porting, ask "is this 80% new or 80%
  existing?"; if 80% new, rewrite; if 80% existing, strip carefully.
- **Cross-agent tool-name mapping** isn't a build-time install step —
  it's a runtime loading step inside each agent. The FORMAT.md doc had
  this wrong.
- **Stub transparency beats stub deception** — 25 honest stubs that
  point at upstream methodology > 36 fake-stubs that pretend to work.
  Production users can differentiate; the difference shows in audit.
- **Re-copies to /tmp** on Windows MSYS can silently fail. Pre-create
  the dir; verify with `ls`; or use `cp -r` for moves, `git clone` for
  fresh sources.
- **The reel's claims** about FB Marketplace automation, pay-per-call
  APIs, and "anything 15% under market" were net-negative — but
  *reframing the goal* (NHTSA vPIC is free, Craigslist is ToS-friendly,
  comps-style local blend) produced a working pipeline.

## Pattern watch list

- Vendor pricing pages: any "free tier" claim should be exercised
  per-endpoint before trusting it (CarsXE lesson).
- Cross-agent migrations: tool name canonical mapping is runtime,
  not install. Don't repeat that mistake.
- Porting with auto-trim: budget for rewriting X% of bodies cleanly
  rather than patching `[PORT-REMOVED]` later.
